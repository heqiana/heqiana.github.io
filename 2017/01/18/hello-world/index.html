    2.变量与变量作用域

    • 全局变量

    ​ window：每个JavaScript环境有一个全局对象，当你在任意的函数外面使用this的时候也可以访问到。

    ​

    当声明一个全局变量时，在全局作用于下可以通过window进行访问，同时也可以通过this访问，并且在全局作 用域下this与window不管是值还是类型都是全等的。

    ​ 尽可能少的使用全局变量，因为可能出现命名冲突导致后面的变量覆盖前面的变量。

    • 隐式全局变量

      其中sum为隐式全局变量，在函数中最好将sum采用var sum=a+b或者let sum=a+b（这里let用法类似于var，但是所声明的变量，只在let命令所在的代码块内有效）将其声明为局部变量，否则可能会造成不必要的麻烦。

    补充（let）：ES6中新增了let命令，用来声明变量。它的用法类似于var，但是所声明的变量，只在let命令所在的代码块内有效。

    第二种隐式全局变量通常采用链式声明变量的情况下，其中b为全局，原因是变量赋值是从右向左，因此首先是给变量b赋值为0。

    所以在进行声明变量时做好采用单var形式，这样做的好处在于可以防止逻辑错误（所有未初始化但声明的变量的初始值是undefined）和增加代码的可读性，同时也帮助自己记住所声明的全局变量，能够一目了然。

    • 隐式全局变量和明确定义的全局变量的区别

      通过var创建的全局变量（任何函数之外的程序中创建）是不能被删除的。无var创建的隐式全局变量（无视是否在函数中创建）是能被删除的。如下所示：

      因此，从上可以看出隐式全局变量并不是真正的全局变量，但它们是全局对象的属性。属性是可以通过delete操作符删除的，而变量是不能的。

    • 变量声明提升

    在函数外面声明了全局name，然后在函数里面声明了局部变量name进行覆盖全局变量，但是其中第一个输出值为未定义，其原因是在函数内部进行重新定义局部变量name，并且其声明会置顶到函数顶部，而赋值在打印输出之后，这种行为称为hoisting（悬置/置顶解析/预解析）。

    补充（代码处理的两个阶段）：代码处理分两个阶段，第一阶段是变量，函数声明，以及正常格式的参数创建，这是一个解析和进入上下文 的阶段。第二个阶段是代码执行，函数表达式和不合格的标识符（为声明的变量）被创建。

    揭秘命名函数表达式

    1．函数表达式和函数声明区别

    函数声明必须带有标示符（Identifier）（就是大家常说的函数名称），而函数表达式则可以省略这个标示符。

    函数声明:

    function 函数名称 (参数：可选){ 函数体 }

    函数表达式：

    function 函数名称（可选）(参数：可选){ 函数体 }

    函数声明只能出现在程序或函数体内。从句法上讲，它们 不能出现在Block（块）（{ … }）中，例如不能出现在 if、while 或 for 语句中。因为 Block（块） 中只能包含Statement语句， 而不能包含函数声明这样的元素。

    2．函数语句

    函数语句不是在变量初始化期间声明的，而是在运行时声明的——与函数表达式一样。不过，函数语句的标识符一旦声明能在函数的整个作用域生效了。

    3．命名函数表达式

    命名函数表达式，理所当然，就是它得有名字，例如var f= function foo(){};就是一个有效的命名函数表达式，但有一点需要记住：这个名字只在新定义的函数作用域内有效。
