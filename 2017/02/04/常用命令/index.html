    git项目部署

    1.从远程仓库克隆到本地

    2.在本地新建分支 XXX，并在分支 XXX 上进行开发，开发完成以后 push 到远程仓库

    rake-zbj 开发流程

    开发新项目

    新项目在 git.zhubajie.la 上，登录 git.zhubajie.la 创建一个新的代码仓库，项目组选择 fe， 项目命名为 fis-{name}，假设我们的新项目名为 fis-jinrong，{name} 为项目代号，同时会作为我们的项目的命名空间，方便跨模块定为资源

    在 workspace 目录下运行 git clone git.zhubajie.la/fe/fis-jinron

    运行完成后，工作目录如下所示

    workspace

    ├── fis-common
    ├── fis-jinrong

​ 新项目 checkout 出来后，运行 rk init 初始化我们的项目，项目ID即为我们的 {name}项目代号

​ 如果 rk init 没有反应，运行 rk -v 看看 rake-zbj 的版本，需要大于 0.2.20

RAP 项目建立

​ 前后端的接口文档，是存放在 rap 网站上的。接口是前后端沟通的桥梁，我们在完成需求分享和模块划分 后，就需要和后端一起确定相关功能接口。
​ 接口定好后，可以将接口录入到 rap 系统中。接口数据录入后，即可通过 rk data 命令一键下载测试数据到本地，方便本地开发和调试。
​ 每次 RAP 系统接口有更新后，都需要运行 rk data 获取最新的测试数据
​ 使用rk data后会新生成测试数据test（里面的数据和page下的模板是同名依赖关系）和路由文件server.conf，注意先在fis-config.js里面确定好rapId与namespace，rapId与RAP上建立的项目id保持一致，不然拉不下来数据。

RAP 使用说明

​ 在 rap 建立好接口后，或者更新接口后，都需要运行 rk data 下载模拟数据
​ 进入 fis-common 目录，运行 rk release -cw，发布 fis-common 模块代码。fis-common为公共组件，通常情况不会发生变化，一般情况不需要加上监听参数w，直接使rk release -c就可以了。
​ 进入 fis-jinrong 目录，运行 rk release -cw，发布 fis-jinrong 模块代码
运行 rk server start，打开本地测试环境地址，输入 fis-jinrong 目录下的 server.conf 文件里相应的接口地址即可 看到效果
​ 发布后的代码在哪里？运行 rk server open 即可看到本地的调试服务器目录
​ 如果发生了调试运行的时候发现即使url变了，但渲染的内容依然是旧的，这就是缓存在作怪，可以rk server open，手动删除缓存；也可以直接使用rk server clean清空目录，然后重新发布一下。

和后端联调

​ 本地开发完后，我们需要和后端进行联调，一般情况下是将代码发布到后端的测试机器上去。
后端要做的事情

​ 让后端将此接收代码的脚本部署到其测试机器上，首先用switchhost切换到后端机子ip。

​ 修改 fis-conf.js，添加 deploy 信息

rapId: 149, // 自己的rapId
namespace: "campus", // 自己的项目名称
rapUrl: 'http://192.168.142.49:3000/?project_id=',
roadmap : {
    domain : "http://a.dev.zbjdev.com"
},
deploy: {
        remote_rd: [
            {
                receiver : receiverUrl,
                from : '/static',
                to : '/{后端 web 目录}/rake-static', // 后端 web 目录，即值 SlightPHP 的上级目录，如果是 windows 系统，用正斜线，即 E:/path/to/webroot
                exclude : /.*\.(?:svn|cvs|tar|rar|psd).*/
            },
            {
                receiver : receiverUrl,
                from : '/config',
                to : '/{后端 web 目录}/rake-zbj',
                exclude : /.*\.(?:svn|cvs|tar|rar|psd).*/
            },
            {
                receiver : receiverUrl,
                from : '/template',
                to : '/{后端 web 目录}/rake-zbj',
                exclude : /.*\.(?:svn|cvs|tar|rar|psd).*/
            },
            {
                receiver : receiverUrl,
                from : '/plugin',
                to : '/{后端 web 目录}/rake-zbj',
                exclude : /.*\.(?:svn|cvs|tar|rar|psd).*/
            }
        ]
        ..可以添加多个后端机器
}

});
roadmap : {
domain : “http://a.dev.zbjdev.com“
},
运行 rk release -Dd remote_rd，如果有错误，可以多运行几次。代码发发布完后，即可访问后端机器看到运行情况
